GGFW Clean 5.5.0 - Stage 10A notification pre-integration

Base: ggfw_clean55_stage1_5_hotfix3_read_delete_wrap_vscode

Added/changed:
- Status bar envelope icon in the former CL/OP/MO/DWR area when Messenger has unread inbox messages.
- Existing status indicator is preserved when there is no unread message.
- Inbox unread flag already existed; status bar now reflects it.
- Opening an Inbox message marks it read and refreshes status line.
- Fake/test inbox message triggers unread icon and optional double-beep.
- RF/FSK code is still not included in this build.

How to test:
1. Build Fusion with VS Code task or bash compile-with-docker.sh Fusion.
2. Confirm normal RX/TX/menu/channel changes remain unchanged.
3. Open Messenger with F+MENU.
4. Press KEY_1 on Messenger home or use Settings -> TESTMSG.
5. Exit Messenger and confirm envelope icon appears in status bar.
6. Re-open Messenger -> Inbox -> open the message.
7. Exit and confirm envelope disappears after message is marked read.
8. Toggle BEEP in Settings and confirm TESTMSG only beeps when BEEP is ON.
